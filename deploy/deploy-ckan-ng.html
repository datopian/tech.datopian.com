    # Deploy A Frontend APP To CKAN Cloud

    # Creating CKAN Portal - Deploying for first time

    In order to deploy CKAN portal on our Kubernetes cluster, you will need to first dockerize the CKAN application. For that you will need:

    • Dockerfile to build an image.
    • .env file to keep environment variables for an image.
    • .gitlab-ci.yml to deploy an image to container registry.

    Later, Kubernetes will pull that image and build a portal using it.

    We keep those file in private GitLab Repositories.

    Create a new project in GitLab:

    • Navigate to https://gitlab.com/viderum (opens new window)
    • Click the ‚ÄúNew Project‚Äù button on the right
    • Let‚Äôs name it cloud-deploy-demo-exercise-ng
      • We use following name convention: cloud-[project-id]-ng
    • Click ‚ÄúCreate Project‚Äù

    You should have an empty repository on https://gitlab.com/viderum/cloud-deploy-demo-exercise-ng (opens new window). Let‚Äôs now add the necessary files.

    # Create Dockerfile

    Dockerfile is a text document that contains all the commands a user could call on the command line to assemble an image. Usually, all you need is to copy the Dockerfile from the existing project, and update its API_URL variable.

    • Click ‚ÄúNew File‚Äù button
    • Copy everything from https://gitlab.com/viderum/cloud-deploy-demo-ng/blob/master/Dockerfile (opens new window) and paste it in your new file
    • Name the file Dockerfile
    • Click ‚ÄúCommit Changes‚Äù button

    Note:

    • Please define All environment variables here as shown in example ENV API_URL https://deploy-demo.ckan.io/api/3/action/

    # Create .env file

    Please create this file as a placeholder for now. It will not be used when deployd. Define all your environment variables in Docerfile

    • Click + and choose ‚ÄúAdd File‚Äù
    • Copy everything from https://gitlab.com/viderum/cloud-deploy-demo-ng/blob/master/.env (opens new window) and paste it in the new file
    • Name the file .env
    • Click ‚ÄúCommit Changes‚Äù button

    # Create .gitlab-ci.yml

    .gitlab-ci.yml is responsible for running continuous deployments. It builds the newest image of the portal and pushes it to the GitLab registry. To create it:

    • Go to https://cc-p-jenkins.ckan.io/job/Initialize Gitlab/ (opens new window)
    • Click Build with Parameters button
    • Change viderum/cloud-instance-id to viderum/cloud-deploy-demo-exercise-ng
    • Click Build button

    At this point you should have all 3 files in your repository.

    Wait until the GitLab Build is successful.

    Green means build was successful, and our image was deployed to GitLab Container registry. That means we are ready to create a new instance on the K8 cluster. üöÄ

    # Create Frontend APP

    Now we need to deploy the new image to CKAN Cloud

    • Go to https://cc-p-jenkins.ckan.io/job/create frontend instance/ (opens new window)
    • Click Build With Parameters button
    • Update INSTANCE_NAME and GITLAB_REPO inputs per your needs
    • Click Build button

    Wait until the build is successful. Once done, that‚Äôs it! Our instance is deployed to the K8 cluster. Finally, let‚Äôs create a route for it.

    # Create route

    In the last step, we need to create the route for your portal, so that it‚Äôs publicly accessible.

    • Go to https://cc-p-jenkins.ckan.io/job/create route (opens new window)
    • Click Build With Parameters button
    • Update EXTRA_ARGS with following: instances-default deploy-demo-exercise-ng deploy-demo-exercise-ng
    • Set DRY_RUN to no
    • Click Build button

    # Check it is working

    That‚Äôs It! üéÜ You should now be able to visit https://cloud-deploy-demo-ng-exercise.ckan.io (opens new window) and see the live portal.
